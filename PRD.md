## 1. 产品概述

### 1.1 产品定位

<!--
  一句话定位：Swap Protection Agent 是什么，为谁解决什么问题
-->

**Swap Protection Agent** 是一款面向 DeFi 交易者的链上 swap 分析工具。在用户通过钱包确认交易之前，提供 **DEX 最优路径推荐**、**预估滑点** 以及 **MEV 风险等级评估**，帮助用户做出更明智的决策，避免因信息不对称而遭受损失。

### 1.2 核心价值主张

<!--
  三个差异化价值点
-->

| 价值 | 说明 |
|------|------|
| **交易前感知风险** | 不等交易发出就能看到潜在 MEV 风险和滑点预估 |
| **DEX 路径透明化** | 跨 DEX 聚合报价，一眼看出哪个交易所最优 |
| **零门槛使用** | 无需安装钱包、无需 Gas，输入参数即可获得分析结果 |

### 1.3 解决的痛点

<!--
  三个被验证的 DeFi 用户痛点
-->

| 痛点 | 现状 | 本产品的解决方案 |
|------|------|------------------|
| **路径选择盲目** | 多数用户在 Uniswap 和 1inch 之间凭经验选择，缺乏数据支撑 | 给出具体的最优 DEX 建议 |
| **滑点估算粗糙** | 钱包默认的滑点容忍度（如 0.5%）无法反映真实链上流动性 | 依据链上状态给出动态滑点预估 |
| **MEV 难以感知** | 用户通常只在被三明治攻击后才在 Etherscan 上发现异常 | 交易前给出 MEV 风险等级预警 |

## 2. 目标用户

### 2.1 用户画像

<!--
  定义核心用户群体及其特征
-->

| 画像 | 特征 | 核心诉求 |
|------|------|----------|
| **Retail Trader（散户交易者）** | 每周进行 5-20 笔 swap，交易额 $100-$10,000，关注 Gas 和滑点成本 | 省心、省 Gas、减少损失 |
| **DeFi Power User（高级玩家）** | 熟练使用 1inch、CowSwap 等聚合器，关注 MEV 和最优执行 | 更精确的风险评估和路径推荐 |
| **黑客松评委 / 投资人** | 技术背景，关注产品创意、架构合理性和可扩展性 | 理解产品逻辑和差异化 |

### 2.2 非目标用户

<!--
  明确哪些用户不在当前服务范围内，避免功能蔓延
-->

- **合约开发者**：本产品不提供合约安全审计功能
- **流动性提供者（LP）**：本产品专注于 swap 交易者视角，非 LP 策略优化
- **机构做市商**：高频、大额交易场景超出 MVP 能力范围

### 6.2 技术选型理由

<!--
  每个技术选型的原因
-->

| 技术 | 原因 |
|------|------|
| **SE-2 (Scaffold-ETH 2)** | 开箱即用的 dApp 脚手架，集成 RainbowKit + Wagmi + Hardhat，适合黑客松快速搭建 |
| **Express + TypeScript** | 轻量、成熟、适合原型阶段；TypeScript 保证类型安全，减少联调 bug |
| **Next.js App Router** | SE-2 默认选择；支持服务端组件，性能好于 Pages Router |
| **DaisyUI + Tailwind** | DaisyUI 提供语义化组件（btn、card、badge），减少 CSS 编写量 |
| **Yarn Workspaces** | Monorepo 原生支持，无需额外工具（Nx/Turborepo） |
| **Mock Data** | MVP 阶段不依赖外部链上 RPC，确保 Demo 稳定可控 |

## 7. API 规范

### 7.1 通用约定

<!--
  API 约定：Base URL、Content-Type、错误格式
-->

| 项 | 约定 |
|----|------|
| Base URL | `http://localhost:4000` |
| Content-Type | `application/json` |
| 错误响应格式 | `{ "error": "错误描述" }` |
| 成功 HTTP 状态码 | `200 OK` |
| 客户端错误码 | `400 Bad Request` |
| 服务端错误码 | `500 Internal Server Error`（预留） |

### 7.2 端点详情

#### `GET /`

**用途**：健康检查，判断 Agent 服务是否运行。

**响应**：

```
HTTP/1.1 200 OK
Content-Type: text/html; charset=utf-8

Agent Running
```

---

#### `POST /analyze`

**用途**：分析 swap 交易，返回最优 DEX、滑点预估、MEV 风险等级。

**请求体**：

```json
{
  "tokenIn": "ETH",
  "tokenOut": "USDC",
  "amount": 1
}
```

**字段说明**：

| 字段 | 类型 | 必填 | 约束 | 示例 |
|------|------|------|------|------|
| `tokenIn` | `string` | ✅ | 非空字符串 | `"ETH"` |
| `tokenOut` | `string` | ✅ | 非空字符串 | `"USDC"` |
| `amount` | `number` | ✅ | 正数（> 0） | `1` |

**成功响应 (200)**：

```json
{
  "bestDex": "Uniswap",
  "slippage": "0.4%",
  "mevRisk": "LOW"
}
```

**响应字段说明**：

| 字段 | 类型 | 说明 | 可能值（模拟） |
|------|------|------|---------------|
| `bestDex` | `string` | 推荐的最优 DEX | `"Uniswap"` |
| `slippage` | `string` | 预估滑点（百分比格式） | `"0.4%"` |
| `mevRisk` | `"LOW" \| "MEDIUM" \| "HIGH"` | MEV 风险等级 | `"LOW"` |

**错误响应 (400)**：

```json
{
  "error": "Missing required fields: tokenIn, tokenOut, amount"
}
```

```json
{
  "error": "amount must be a positive number"
}
```

## 10. 后续迭代规划

### 10.1 Phase 路线图

<!--
  从 MVP 到完整产品的分阶段规划
-->

```
Phase 1-3  ──→  Phase 4    ──→  Phase 5    ──→  Phase 6
(MVP ✅)      (链上数据)      (MEV 检测)      (执行引擎)
```

#### Phase 4 — 链上数据集成（下一阶段）

- 接入 DEX 链上报价：Uniswap V3 / Sushi / Curve
- 基于流动性池储备量计算真实滑点
- 支持代币合约地址校验（不只是符号名称）
- 前端 `loading` 时间适配（链上请求可能需 2-3 秒）

#### Phase 5 — MEV 风险检测

- Mempool 监听：检测挂起的 swap 交易
- 三明治攻击识别：分析交易排序模式
- 风险建模：结合交易金额、代币流动性、当前 mempool 拥堵度给出分级

#### Phase 6 — 执行引擎（远期）

- 集成 `wagmi` 交易发送：用户可直接在前端签名执行
- 最优路径路由：自动选择 DEX 聚合器（1inch / CowSwap）
- Flashbots 保护：高风险交易自动走私有 mempool

### 10.2 优先级矩阵

| | 高价值 | 低价值 |
|---|---|---|
| **低投入** | Phase 4：链上报价聚合 | F-FE-08：历史记录 |
| **高投入** | Phase 5：MEV mempool 监控 | F-BE-11：API Key 认证 |

## 11. 风险评估

### 11.1 技术风险

| 风险 | 等级 | 说明 | 缓解措施 |
|------|------|------|----------|
| 链上 RPC 不稳定 | 🟡 中 | DEX 报价依赖 RPC 节点响应速度 | 使用多 RPC 备选 + 本地缓存 + 超时降级 |
| MEV 检测准确度 | 🔴 高 | 静态分析无法完全模拟真实 MEV 环境 | Phase 5 引入 mempool 实时监控 |
| DEX 报价不一致 | 🟡 中 | 链上报价与用户最终成交价可能有偏差 | 在 UI 上明确标注"预估"字样 |
| 多链兼容复杂度 | 🟢 低 | MVP 仅考虑以太坊主网 | 架构预留 chain ID 参数 |

### 11.2 产品风险

| 风险 | 等级 | 说明 | 缓解措施 |
|------|------|------|----------|
| 用户操作门槛 | 🟡 中 | 需要用户了解代币符号和基本 DeFi 概念 | 提供默认示例值（ETH → USDC） |
| 竞品压力 | 🟡 中 | 1inch、CowSwap 等聚合器已提供类似功能 | 差异化定位在"分析"而非"执行" |
| 数据依赖 | 🔴 高 | 产品价值高度依赖链上数据的质量和实时性 | Phase 4 重点打磨数据层 |

## 附录

### A. 术语表

| 术语 | 说明 |
|------|------|
| **DEX** | 去中心化交易所（如 Uniswap、SushiSwap） |
| **MEV** | Maximal Extractable Value，矿工/验证者可提取价值，包含三明治攻击、抢先交易等 |
| **滑点 (Slippage)** | 预期成交价与实际成交价之间的偏差 |
| **三明治攻击** | MEV 攻击的一种，攻击者在用户交易前后插入买卖操作获利 |
| **Flashbots** | 私有交易中继服务，可避免交易在公共 mempool 中暴露 |
| **Monorepo** | 单一代码仓库管理多个 packages 的项目结构 |
| **MVP** | Minimum Viable Product，最小可行产品 |
